# 记忆系统

## 当前状态

Go runtime 的记忆能力已经从早期 remember 文件升级为 SQLite memory store。当前支持 FTS 检索、可选 embedding、observation / fact 生命周期、learn / consolidate / feedback，以及面向 agent 的 memory tools。

`POST /api/remember` 仍作为兼容入口保留；新能力以 agent `memoryConfig` 和 runtime memory store 为主。

## 核心流程

```text
agent memoryConfig.enabled
  -> query 前召回 Stable Memory / Relevant Observations
  -> prompt 注入 memory context
  -> run 完成后可 autoRemember / learn
  -> observation 整理、去重、晋升
  -> memory tools 显式读写和治理
```

memory store 中的长期稳定事实通常作为 `Stable Memory` 注入；近期观察类信息作为 `Relevant Observations` 注入。agent 目录下的 `memory/memory.md` 是静态背景提示，不进入 runtime memory store。

## 配置与接口

- `MEMORY_DIR`：memory 数据目录。
- `AGENT_MEMORY_DB_FILE_NAME`：SQLite 文件名。
- `AGENT_MEMORY_CONTEXT_TOP_N`：query 前召回条数。
- `AGENT_MEMORY_CONTEXT_MAX_CHARS`：注入 prompt 的最大字符数。
- `AGENT_MEMORY_SEARCH_DEFAULT_LIMIT`：memory tools 默认 limit。
- `memoryConfig.enabled`：开启 runtime memory 与基础 memory tools。
- `memoryConfig.managementTools`：注入管理类 memory tools。
- `memoryConfig.autoRemember.enabled`：run 完成后自动 learn。
- `memoryConfig.embedding.providerKey`：启用 provider 侧 embedding。

主要 tools：

- `memory_write`
- `memory_read`
- `memory_search`
- `memory_update`
- `memory_forget`
- `memory_timeline`
- `memory_promote`
- `memory_consolidate`

## 约束与注意事项

- 未配置 `memoryConfig.enabled: true` 的 agent 不启用 runtime memory。
- embedding provider 未配置时，检索降级为 FTS 与重要度排序。
- `memory/memory.md` 是静态提示，不是数据库备份。
- 旧 `MEMORY_*` 环境变量族已被清理，记忆开关以 agent 配置为准。
