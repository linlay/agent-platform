# 会话存储与回放

## 当前状态

Chat 数据以本地文件系统为主，根目录由 `CHATS_DIR` 控制。运行事件、StepLine、raw messages、上传资源、归档与搜索都围绕 chat store 组织。

## 核心流程

```text
query request
  -> ensure chat
  -> stream events
  -> StepWriter 聚合 StepLine
  -> events.jsonl / raw_messages.jsonl
  -> chat summary / archive / replay
```

默认布局：

```text
<CHATS_DIR>/
  index.json
  <chatId>/
    events.jsonl
    raw_messages.jsonl
    <uploaded-file>
```

StepLine 快照中，`_type:"query"` 可携带 system init 快照；`_type:"react".seq` 是 react mode 的模型调用分组 id，不是 JSONL 物理行号。

## 配置与接口

- `CHATS_DIR`：chat 根目录。
- `CHAT_STORAGE_K`：query 时回放的历史 run 窗口。
- `/api/chats`
- `/api/chat`
- `/api/chats/search`
- `/api/chat/export`
- `/api/archives`
- `/api/archive`
- `/api/archives/search`
- `/api/upload`
- `/api/resource`

## 约束与注意事项

- `GET /api/chat` 默认返回 `events`，`includeRawMessages=true` 时附带 `rawMessages`。
- 文件资源落在 chat 目录下，resource ticket 只保护访问，不改变落盘位置。
- `system` 快照首次出现必写，后续仅在 model/messages/tools/stream 变化时写入。
- 历史旧数据由 replay 兼容路径读取，删除兼容逻辑前需参考 [兼容清理清单](兼容清理清单.md)。
