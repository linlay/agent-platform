# 兼容清理清单

## 当前状态

本文记录 Go runtime 中仍有意保留的兼容路径。删除这些路径前，需要先确认代表性本地或生产数据已经不再依赖它们。

## 核心流程

兼容路径清理遵循：

```text
发现兼容路径 -> 文档登记 -> 审计真实数据 -> 保持观察窗口 -> 单独提交删除
```

当前保留项：

- legacy run id 解析仍保留，用于读取旧 chat 数据。
- 单 gateway 旧形态已不再从 env 合成，但相关拒绝和诊断信息仍保留。
- 废弃环境变量采用启动硬失败，而不是静默忽略。
- `SystemInitLegacy` fallback 仍用于历史 chat 回放。
- agent 旧字段如 `sandboxConfig` 仅作为迁移线索处理，新配置应使用 `runtimeConfig`。

## 配置与接口

- 审计脚本入口：`scripts/audit-compat-data.sh <CHATS_DIR> [CONFIG_ROOT]`
- 配置事实源：`internal/config/config.go`
- chat 历史数据事实源：`CHATS_DIR`

## 约束与注意事项

- 不要在同一次提交里同时删除兼容路径和重写主协议，便于回滚。
- 删除前必须确认 docs、tests、runtime 配置样本中没有继续推荐旧字段。
- 废弃 env 的硬失败本身属于兼容治理能力，不能随意改成忽略。
