# 鉴权与安全边界

## 当前状态

服务鉴权以 JWT 为主，支持本地公钥文件与 JWKS。resource ticket 用于保护 `/api/resource` 文件访问。CORS 由 `configs/runtime.yml` 的 `cors` 节控制。敏感配置通过 `.env`、真实 `configs/*.yml`、真实 `configs/*.pem` 或外部 Secret 注入，不提交仓库。

工具级文件和命令权限不在本文展开，见 [工具目录权限](工具目录权限.md)。

## 核心流程

```text
HTTP / WebSocket request
  -> token 提取
  -> 本地公钥或 JWKS 验签
  -> issuer 等声明校验
  -> handler
```

resource 访问在普通鉴权之外还可启用 ticket：

```text
GET /api/resource?file=...&t=...
  -> 校验 resource ticket
  -> 定位 CHATS_DIR 内资源
  -> 返回文件
```

- `configs/runtime.yml`

## 约束与注意事项

- `AUTH_LOCAL_PUBLIC_KEY_FILE` 为相对路径时按项目根目录解析。
- `AUTH_JWKS_URI` 非空时走 JWKS 模式。
- WebSocket 鉴权复用 HTTP token 校验链路。
- provider `apiKey` 按明文字符串读取，不提供加密或解密机制。
- `runtimeConfig.env` 不通过 catalog API 回显。
