# 通讯协议

## 当前状态

运行时同时提供 REST、SSE 与 WebSocket。REST 承载 catalog、chat、resource、memory 等请求；`POST /api/query` 使用 SSE 返回实时 run stream；`/ws` 是控制面，复用请求、响应、stream 与文件引用，不直接承载浏览器文件字节。

## 核心流程

```text
普通 JSON API -> ApiResponse envelope
Query API -> SSE stream -> [DONE]
WebSocket -> request / response / stream / push frames
文件上传下载 -> HTTP 数据面
```

文件传输按“HTTP 数据面 + WebSocket 控制面”划分：浏览器上传走 `POST /api/upload`，下载走 `GET /api/resource`；WebSocket `/api/upload` 只用于网关发送 `url + metadata`，由 platform 自己通过 HTTP 拉取并校验。

## 配置与接口

主要 REST 组：

- Catalog：`/api/agents`、`/api/agent`、`/api/teams`、`/api/skills`、`/api/tools`、`/api/tool`
- Chat：`/api/chats`、`/api/chat`、`/api/chats/search`、`/api/read`、`/api/chat/export`
- Run：`/api/query`、`/api/attach`、`/api/submit`、`/api/steer`、`/api/interrupt`
- Resource：`/api/upload`、`/api/resource`
- Viewport：`/api/viewport`
- Archive：`/api/archives`、`/api/archive`、`/api/archives/search`

## 约束与注意事项

- 非 SSE JSON 接口统一返回 `code/msg/data`。
- `GET /api/attach`、`POST /api/submit`、`POST /api/steer`、`POST /api/interrupt` 都需要 `agentKey`。
- WebSocket 鉴权复用 HTTP token 校验链路。
- 反向 gateway 配置在 `configs/channels.yml`，不再通过旧单 gateway env 合成。
